# AI Leadership Insight Agent

> **Task 1 — AI Leadership Insight & Decision Agent**

An AI-powered assistant that helps leadership understand the current status, performance, and strategic position of the organization — grounded entirely in internal company documents.

> **For a quick demo and testing, see the Colab notebook:** [`demo.ipynb`](demo.ipynb)

---

## What This Solves

Leadership teams need fast, reliable answers from internal documents — annual reports, quarterly updates, strategy notes, operational reviews. This system lets you ask natural-language questions and get factual, document-grounded answers in seconds, with optional auto-generated charts for data-heavy questions.

---

## Key Design Decisions

- **LLM chunking** — Gemini splits documents into semantically coherent passages (400–700 tokens) rather than fixed token windows, preserving meaning across chunk boundaries
- **Question-augmented embeddings** — 4–5 questions are generated per chunk and embedded alongside it, dramatically improving retrieval recall for leadership-style queries
- **Grounded generation** — the answer prompt explicitly instructs Gemini to use only the retrieved context, preventing hallucination
- **Auto chart detection** — Gemini decides whether a chart would help answer the question; if yes, it extracts structured data from context and renders it via matplotlib (data is validated before plotting — no hallucinated numbers)
- **Multi-tenant isolation** — every document is tagged with `client_id` + `project_id`, so searches are always scoped to the right organisation and project

---

## Technical Overview

A production-ready Retrieval-Augmented Generation (RAG) system that ingests PDF documents into Zilliz Cloud (Milvus), and answers natural language questions from the stored knowledge — with optional auto-generated charts.

---

## How It Works

```
CREATION PIPELINE
PDF  →  PyMuPDF  →  Gemini LLM Chunker  →  Gemini Q-Gen  →  Gemini Embeddings (llm chunk + Question generated by Gemini)  →  Zilliz Cloud

CONSUMPTION PIPELINE
Question  →  Gemini Embeddings  →  Zilliz COSINE Search  →  Gemini LLM Answer  +  matplotlib Chart
```

## Setup

### 1. Clone the repository

```bash
git clone https://github.com/chetanabanerjee09/ai-leadership-insight-agent.git
cd ai-leadership-insight-agent
```

### 2. Create a conda environment

```bash
conda create --name rag_project python=3.10
conda activate rag_project
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Set environment variables

These are **required**:

```bash
export GEMINI_API_KEY=your_gemini_api_key_here
export MILVUS_URI=your_zilliz_milvus_uri
export MILVUS_TOKEN=your_zilliz_milvus_token
```

### 5. Zilliz Cloud Setup

1. Sign up at [cloud.zilliz.com](https://cloud.zilliz.com) — free serverless tier available
2. Create a **Serverless** cluster
3. Go to your cluster → **Connect** → copy the **Public Endpoint** and **API Key**
4. Set them as `MILVUS_URI` and `MILVUS_TOKEN` (step 3 above)
5. **Make sure your cluster is Running** — serverless clusters auto-pause after inactivity; resume from the Zilliz console if needed

## Running the Creation Pipeline (Ingest Documents)

Place your PDF files in the `inputs/` folder, then run:

```bash
python creation_pipeline/infer_creation.py \
  --files inputs/adobe-2024-annual-report.pdf \
  --client_id adobe \
  --project_id annual_report
```

**Multiple files at once:**
```bash
python creation_pipeline/infer_creation.py \
  --files inputs/report.pdf inputs/strategy.pdf \
  --client_id adobe \
  --project_id annual_report
```

**Custom collection name:**
```bash
python creation_pipeline/infer_creation.py \
  --files inputs/report.pdf \
  --client_id adobe \
  --project_id annual_report \
  --collection my_collection
```

### Creation Pipeline Arguments

| Argument | Required | Default | Description |
|---|---|---|---|
| `--files` | Yes | — | One or more local PDF paths |
| `--client_id` | Yes | — | Logical client identifier (used as a filter in Milvus) |
| `--project_id` | Yes | — | Logical project identifier (used as a filter in Milvus) |
| `--collection` | No | `rag_collection` | Override the Milvus collection name |

### Expected Output

```json
{
  "status": "success",
  "rows_inserted": 1240,
  "chunk_records": 248,
  "files_processed": 1,
  "collection": "rag_collection",
  "client_id": "adobe",
  "project_id": "annual_report"
}
```

> **Note:** The collection is auto-created on first run with the correct schema and COSINE vector index. Subsequent runs on the same collection are safe — it skips recreation.

## Running the Consumption Pipeline (Ask Questions)

Use the **same** `client_id` and `project_id` as ingestion:

```bash
python consumption_pipeline/infer_consumption.py \
  --question "What was Adobe's total revenue in 2024?" \
  --client_id adobe \
  --project_id annual_report
```

**With a chart (auto-decided):**
```bash
python consumption_pipeline/infer_consumption.py \
  --question "Show me Adobe's revenue trend across 2024 quarters" \
  --client_id adobe \
  --project_id annual_report \
  --plot_dir plots
```

**Save retrieved context for debugging:**
```bash
python consumption_pipeline/infer_consumption.py \
  --question "What are Adobe's main product segments?" \
  --client_id adobe \
  --project_id annual_report \
  --save_context
```

### Consumption Pipeline Arguments

| Argument | Required | Default | Description |
|---|---|---|---|
| `--question` | Yes | — | The question to answer (in quotes) |
| `--client_id` | Yes | — | Must match what was used during ingestion |
| `--project_id` | Yes | — | Must match what was used during ingestion |
| `--collection` | No | `rag_collection` | Override the Milvus collection name |
| `--plot_dir` | No | `plots` | Directory to save auto-generated charts |
| `--save_context` | No | `False` | Save retrieved chunks to `retrieved_context.txt` |

### Expected Output

```
============================================================
Question : How is the workforce distributed between the U.S. and international locations?
============================================================
Answer   : As of November 29, 2024, 50% of the workforce was in the United States and 50% was in international locations.
Plot     : /Users/you/RAG/plots/workforce_distribution.png
============================================================
```

## Configuration

All non-sensitive settings live in `configs/config.yaml`:

```yaml
gemini:
  chunking_model: gemini-2.0-flash-001     # LLM used for semantic chunking
  question_model: gemini-2.0-flash-001     # LLM used to generate questions
  embedding_model: gemini-embedding-001    # Embedding model
  generation_model: gemini-2.0-flash-001  # LLM used to generate answers
  embedding_dim: 3072                      # Must match your Milvus collection dim
  max_concurrent_requests: 30
  embedding_batch_size: 50

milvus:
  collection_name: rag_collection          # Default collection name
  batch_size: 500                          # Insert batch size

retrieval:
  top_k: 20                                # Number of chunks retrieved per query
```

## Milvus Collection Schema

The collection is auto-created by the ingestion pipeline. Schema for reference:

| Field | Type | Details |
|---|---|---|
| `id` | INT64 | Auto-increment primary key |
| `text` | VARCHAR | Chunk passage text |
| `question` | VARCHAR | One generated question per row |
| `dense_vector` | FLOAT_VECTOR | dim=3072, COSINE / IVF_FLAT index |
| `client_id` | VARCHAR | Filter field — set at ingestion time |
| `project_id` | VARCHAR | Filter field — set at ingestion time |
| `page_number` | INT64 | Source PDF page number |

> Each chunk produces **4–5 rows** in Milvus (one per generated question). The retriever deduplicates them so each chunk text appears only once in the final context.

## Troubleshooting

| Error | Cause | Fix |
|---|---|---|
| `MilvusException: Fail connecting to server` | Wrong URI/token or cluster is paused | Check `MILVUS_URI` / `MILVUS_TOKEN`; resume cluster in Zilliz console |
| `EnvironmentError: GEMINI_API_KEY is not configured` | Env var not set | `export GEMINI_API_KEY=...` |
| `ImportError: matplotlib is required` | matplotlib not installed | `pip install matplotlib` |
| `ImportError: PyMuPDF is required` | pymupdf not installed | `pip install pymupdf` |
| `No relevant chunks found` | Wrong client_id/project_id or docs not ingested | Use same IDs as ingestion; run creation pipeline first |
| `Dimension mismatch` | Collection was created with a different embedding dim | Delete the collection and re-ingest, or update `embedding_dim` in config.yaml |
